# Study

这不是“前端教程”，而是一套围绕当前项目的学习脚手架。

目标只有一个：让一个主要懂 Python 的人，能够用最少的前端知识看懂这个项目是怎么跑起来的。

建议按下面顺序学习，不要跳着看：

1. 先看 [01_知识点学习列表.md](./01_知识点学习列表.md)
2. 再看 [02_一次点击的完整链路.md](./02_一次点击的完整链路.md)
3. 然后看 [03_前端最小必要知识.md](./03_前端最小必要知识.md)
4. 接着看 [04_后端与业务核心.md](./04_后端与业务核心.md)
5. 最后用 [05_模仿模板.md](./05_模仿模板.md) 自己复盘

## 学习原则

每看一个文件，只回答 4 个问题：

1. 这个文件的输入是什么
2. 这个文件的输出是什么
3. 谁调用它
4. 它再调用谁

不要一上来追求“全懂语法”。你真正要建立的是“调用链”和“数据流”。

## 这个项目最适合你的切入方式

先把它看成一条流水线：

`网页按钮 -> 前端请求 -> Python 后端接口 -> Python 业务处理 -> 本地文件输出`

你现在最重要的不是学会 React，而是先学会在代码里追踪这条线。

## 先记住这几个角色

- `frontend/src/App.tsx`
  - 页面总控，负责按钮、状态、展示结果
- `frontend/src/api.ts`
  - 前端请求后端的地方
- `backend/app/main.py`
  - 本地 HTTP 服务入口
- `backend/app/services/extract_service.py`
  - 抽取主流程
- `backend/app/services/render_service.py`
  - Word 渲染主流程

## 学完后的标准

如果你能自己口头讲清楚下面这三句话，就说明已经入门了：

1. 为什么这个项目需要前端
2. 点击“执行抽取”以后，代码按什么顺序跑
3. 最终生成的 Word 文件是由哪几个 Python 文件配合产生的
