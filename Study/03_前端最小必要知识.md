# 前端最小必要知识

这一份只讲“够看懂这个项目”的前端知识，不追求系统学习。

## 1. 先用一句话理解前端

前端就是：

- 把页面显示出来
- 接住你的点击、输入、上传
- 把这些内容发给后端
- 再把后端结果显示回来

在这个项目里，前端不是核心业务处理者，它更像“网页版控制面板”。

## 2. React、TypeScript、Vite 最直白的解释

### React

React 是一种写网页界面的方式。

你可以把它理解成：

- “页面由很多小组件拼起来”
- “状态变了，页面就跟着更新”

在这个项目里：

- `frontend/src/App.tsx`
  - 总控组件
- `frontend/src/components/FileInputPanel.tsx`
  - 上传区
- `frontend/src/components/SettingsPanel.tsx`
  - 设置区
- `frontend/src/components/OutputPanel.tsx`
  - 输出区

### TypeScript

TypeScript 是前端代码语言。

你可以先把它理解成：

- “前端版的 Python + 更严格的类型检查”

在这个项目里：

- `frontend/src/types.ts`
  - 定义设置对象、接口返回对象、日志对象的结构

### Vite

Vite 是前端开发工具。

它负责：

- 启动前端开发服务
- 把前端代码编译给浏览器运行
- 把 `/api` 请求转发给 Python 后端

看这里：

- `frontend/vite.config.ts`

## 3. 前端项目里最该看的文件

### `frontend/src/main.tsx`

作用：

- 把整个前端应用挂载到页面上

你把它理解成：

- 前端入口

### `frontend/src/App.tsx`

作用：

- 管总状态
- 管按钮事件
- 管结果展示

你把它理解成：

- 页面主程序

### `frontend/src/api.ts`

作用：

- 请求后端

你把它理解成：

- 前端版的 `requests.py`

### `frontend/src/types.ts`

作用：

- 约束对象长什么样

你把它理解成：

- 集中定义数据结构

## 4. 在这个项目里你可以先忽略什么

为了不把自己淹没，前期可以先不深究：

- 组件的样式细节
- `Ant Design` 的完整用法
- React 的所有 Hook 原理
- TypeScript 的高级类型语法

你前期只需要知道：

- 状态放在哪
- 点击按钮去哪
- 请求怎么发
- 结果怎么显示

## 5. 你最需要建立的前端思维

不是“React 会不会写”，而是：

1. 页面状态放在哪里
2. 用户动作由哪个函数响应
3. 这个函数调用哪个后端接口
4. 后端返回后，页面哪些状态会被刷新

如果你每次都按这 4 个问题去看前端代码，就不会再觉得前端是“魔法”。

## 6. 结合本项目的一个最小例子

用户点击“执行抽取”：

1. `App.tsx` 里的 `handleExtract` 被调用
2. 它调用 `api.ts` 里的 `extractCaseData`
3. `api.ts` 发 `POST /api/extract`
4. Python 后端处理
5. 结果返回给 `App.tsx`
6. 页面状态更新
7. OCR 文本、抽取结果、replace_map 出现在页面上

这就是你现在最应该掌握的前端知识。
