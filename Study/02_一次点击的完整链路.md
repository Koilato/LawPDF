# 一次点击的完整链路

这一份只做一件事：教你如何追踪“执行抽取”。

不要分心去看所有文件。只看下面这 4 个。

- `frontend/src/App.tsx`
- `frontend/src/api.ts`
- `backend/app/main.py`
- `backend/app/services/extract_service.py`

## 你要模仿的学习动作

每到一个文件，都写下：

1. 这个文件在这一条链里负责什么
2. 它收到了什么
3. 它产出了什么
4. 下一跳去哪里

## 第 1 站：`frontend/src/App.tsx`

重点看：

- `handleExtract`
- `handleRender`
- 页面状态定义

你把它当成：

- 页面主程序
- 用户点击按钮后的第一响应者

你重点回答：

1. 点击“执行抽取”时，是哪个函数开始执行
2. 它会先检查什么
3. 它调用了哪个 API 函数
4. 它拿到结果后，把哪些状态更新了

你可以自己写成这种伪代码：

```python
def handle_extract():
    检查文件有没有选
    result = 调后端接口
    把 result 里的 markdown/json/replace_map 显示到页面
```

## 第 2 站：`frontend/src/api.ts`

重点看：

- `fileToBase64`
- `serializeFile`
- `extractCaseData`

你把它当成：

- 前端版请求客户端

你重点回答：

1. 浏览器里的文件是怎么变成可发送内容的
2. 为什么要 base64
3. 它请求的是哪个 URL
4. 发出去的 JSON 大概长什么样

你可以自己写成这种伪代码：

```python
def extract_case_data(files, config_text, case_name):
    payload = {
        "case_name": case_name,
        "files": {
            "enterpriseReportPdf": 序列化后的文件,
            "lawyerLetterPdf": 序列化后的文件,
        },
        "replace_map_config_text": config_text,
    }
    return post("/api/extract", json=payload)
```

## 第 3 站：`backend/app/main.py`

重点看：

- `do_POST`
- `_handle_extract_request`
- `_write_upload`

你把它当成：

- 后端接口入口
- 请求分发器

你重点回答：

1. `/api/extract` 是在哪里被分发的
2. 收到 base64 后，后端做的第一件事是什么
3. 上传文件被写到哪个目录
4. 真正的抽取主函数是谁

你可以自己写成这种伪代码：

```python
def _handle_extract_request(payload):
    把 enterprise pdf 写到磁盘
    把 lawyer pdf 写到磁盘
    读取运行时设置
    extract_result = 调 extract_service
    再生成 replace_map
    return 最终响应
```

## 第 4 站：`backend/app/services/extract_service.py`

重点看：

- `prepare_case_dirs`
- `trim_pdf_last_page`
- `ocr_pdf_to_markdown`
- `extract_case_data`

你把它当成：

- 抽取总流水线

你重点回答：

1. 案件目录结构是怎么建出来的
2. OCR 在哪一步发生
3. 企业报告和律师函分别由谁处理
4. `logical` 在哪里生成

你可以自己写成这种伪代码：

```python
def extract_case_data(...):
    建目录
    复制输入文件
    如有需要裁掉律师函最后一页
    PDF -> markdown
    markdown -> Defandent
    markdown -> DemandLetter
    Defandent + DemandLetter -> logical
    return 所有结果
```

## 一条完整的人话调用链

你最后要能自己说出这段话：

“我在页面点执行抽取后，`App.tsx` 会调用 `api.ts`。`api.ts` 会把浏览器里的文件转成 base64 并请求 `/api/extract`。后端 `main.py` 收到请求后先把文件写回磁盘，再调用 `extract_service.py` 的主流程。这个主流程会做 OCR、企业报告抽取、律师函抽取和 logical 生成，最后把结果返回给前端显示。”

## 自测题

如果你能回答下面 5 个问题，就说明这一份学会了：

1. 为什么 `App.tsx` 不是“真正干活”的地方
2. 为什么 `api.ts` 需要 `fileToBase64`
3. 为什么 `main.py` 要先 `_write_upload`
4. 为什么 `extract_service.py` 里要先建目录再处理文件
5. `Defandent`、`DemandLetter`、`logical` 分别是哪一步生成的
